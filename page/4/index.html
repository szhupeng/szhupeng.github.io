Android Loaders

发表于 2016-10-17 | 分类于 Android | | 阅读次数:

Android 3.0中引入了加载器，使得在Activity或Fragment中异步加载数据变得更加容易。 加载器具有以下特征：

  • 可用于每个Activity和Fragment
  • 支持异步加载数据
  • 监控其数据源并在内容变化时传递新结果
  • 在某一配置更改后重建加载器时，会自动重新连接上一个加载器的Cursor

因此，它们无需重新查询其数据。

Android使用NFC共享文件

发表于 2016-10-14 | 分类于 Android | | 阅读次数:

Android允许你使用Android Beam文件传输功能在设备之间传输大文件。 此功能有一个简单的API，并且允许用户通过简单地触摸设备开始传输过程。 作为响应，Android Beam文件传输会自动将文件从一个设备复制到另一个设备，并在完成后通知用户。

虽然Android Beam文件传输API处理大量数据，但Android 4.0（API级别14）中引入的Android Beam NDEF传输API可处理少量数据（如URI或其他小消息）。 此外，Android Beam只是Android NFC框架中可用的功能之一，它允许你从NFC标签读取NDEF消息。

Android应用程序间文件共享

发表于 2016-10-12 | 分类于 Android | | 阅读次数:

一个应用程序通常有向其他应用程序提供文件的需要。例如，一个图片画廊可能要向图片编辑器提供图片文件，或者一个文件管理应用程序可能希望允许用户在外部存储中的区域之间复制和粘贴文件。发送应用程序可以共享文件的一种方式是响应来自接收应用程序的请求。

任何情况下，你的应用程序共享文件给其他应用程序的唯一安全的方法是向接收应用程序发送文件的内容URI并给该URI授予临时权限。有临时权限的内容URI因为只提供给了接收该URI的应用程序并且能自动过期所以是安全的。android的 FileProvider 组件提供了方法 getUriForFile() 来生成文件的内容URI。

如果你想在应用程序之间共享少量的文本或者数字数据，你应该发送一个包含这些数据的 Intent。

使用LinearLayout的weight属性按比例分配空间

发表于 2016-09-24 | 分类于 Android | | 阅读次数:

首先看看android:layout_weight属性的作用：用来分配属于空间的一个属性，你可以设置它的权重。

如果我们需要在一个LinearLayout里面，使用一个Button，这个Button的宽度要占到整个布局的一半，在XML中，要如何实现呢？

如果使用想写死Button的宽度的话，是不现实的，因为当你的手机设备不同时，屏幕的分辨率就不同了，所以是没办法实现这个效果的！此时，可以使用LinearLayout中的android:weightSum和android:layout_weight属性了。只需要在XML中，为LinearLayout设置属性android:weightSum=”1”，然后在Button中设置属性android:layout_weight=“0.5”，这样，Button将会占到整个布局的1/2，也就是布局宽度的一半了。我们想要的效果就达到了，而且不会因为手机设备不同，而不能适配。

assets和drawable下图片加载的区别

在hdpi/xhdpi/xxhdpi等等不同dpi的文件夹下的图片在不同的设备上会经过scale的处理。例如我们只在hdpi的目录下放置了一张100100的图片，那么根据换算关系，xxhdpi的手机去引用那张图片就会被拉伸到200200。需要注意到在这种情况下，内存占用是会显著提高的。对于不希望被拉伸的图片，需要放到assets或者nodpi的目录下。
