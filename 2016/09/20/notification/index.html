Notification笔记

发表于 2016-09-20 | 分类于 Android | 阅读次数:

Notification作用

  • 显示接收到的短消息，即时通讯消息等
  • 显示APP客户端所推送的消息
  • 显示正在运行的进程，如后台进程，音乐播放后台服务，下载进度等

Android7.0直接回复功能

踩过的坑

最近在做IM方面的应用，免不了要用到Notification，然后在构建PendingIntent的时候是这样的：

PendingIntent.getActivity(context, requestCode, intent, 0);

第一个人发消息过来了，哇，通知也弹出来了，点击也能如期进入对应的聊天节面，然而，第二个人也发消息过来了，通知也有了，问题来了！点击之后怎么不是跳到第二个人的聊天界面，还是在第一个人的聊天界面，于是在聊天界面打印Intent中传递的值，结果不合理啊，怎么一直是第一个Intent的值？

开始各种排查，结果解决办法让我有砸电脑的冲动，先看看getActivity具体参数：

PendingIntent getActivity(Context context, int requestCode, Intent intent, @Flags int flags);

后来发现flags，上面有提到flags的取值：

PendingIntent.getActivity(context, 1, intent, PendingIntent.FLAG_UPDATE_CURRENT);

再试试，解决了！！！
