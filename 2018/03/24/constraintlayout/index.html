ConstraintLayout用法详解

发表于 2018-03-24 | 分类于 Android | 阅读次数:

A ConstraintLayout is a ViewGroup which allows you to position and size widgets in a flexible way.

相对定位（Relative positioning）

相对定位是在ConstraintLayout中创建布局的基本构建块之一。这些约束允许将指定的控件相对于另一个控件进行定位。可以在水平和垂直方向上约束一个控件：

  • Horizontal Axis: left, right, start and end sides
  • Vertical Axis: top, bottom sides and text baseline

以下是可用约束的列表：

  • layout_constraintLeft_toLeftOf
  • layout_constraintLeft_toRightOf
  • layout_constraintRight_toLeftOf
  • layout_constraintRight_toRightOf
  • layout_constraintTop_toTopOf
  • layout_constraintTop_toBottomOf
  • layout_constraintBottom_toTopOf
  • layout_constraintBottom_toBottomOf
  • layout_constraintBaseline_toBaselineOf
  • layout_constraintStart_toEndOf
  • layout_constraintStart_toStartOf
  • layout_constraintEnd_toStartOf
  • layout_constraintEnd_toEndOf

它们都将引用另一个控件的id或parent（它将引用父容器，即ConstraintLayout）：

Margins

  • android:layout_marginStart
  • android:layout_marginEnd
  • android:layout_marginLeft
  • android:layout_marginTop
  • android:layout_marginRight
  • android:layout_marginBottom

注意，侧边距只能为正数或零，并且带尺寸单位

当约束的控件为GONE时的侧边距

  • layout_goneMarginStart
  • layout_goneMarginEnd
  • layout_goneMarginLeft
  • layout_goneMarginTop
  • layout_goneMarginRight
  • layout_goneMarginBottom

设置以上属性，当约束的控件可见性为GONE时，属性值就会生效，否则不会。

居中定位和偏移

ConstraintLayout比较有用的地方是处理“不可能”的约束：

水平居中

条件：

  • android:layout_width属性为wrap_content或固定值
  • app:layout_constraintLeft_toLeftOf=”parent”
  • app:layout_constraintRight_toRightOf=”parent”

注意：如果android:layout_width为0，则相当于match_parent

垂直居中

条件：

  • android:layout_height属性为wrap_content或固定值
  • app:layout_constraintTop_toTopOf=”parent”
  • app:layout_constraintBottom_toBottomOf=”parent”

注意：如果android:layout_height为0，则相当于match_parent

偏移（Bias）

遇到这种相反约束时的默认值是将控件居中; 但可以使用bias属性来调整位置以偏向另一边：

  • layout_constraintHorizontal_bias
  • layout_constraintVertical_bias

环形定位 (在1.1中添加)

可以相对于另一个控件为中心以一定角度和距离约束一个控件的中心。这就可以将控件放置在一个圆上。可以使用以下属性：

  • layout_constraintCircle : 引用另一个控件的id
  • layout_constraintCircleRadius : 到另一个控件中心的距离
  • layout_constraintCircleAngle : 控件应该在哪个角度上 (从0到360)


可见性行为（Visibility behavior）

ConstraintLayout对标记为View.GONE的控件有特定处理。

像往常一样，可见性为GONE的控件不会被显示，并且不是布局本身的一部分（即，如果标记为GONE，其实际尺寸不会被改变）。

但就布局计算而言，可见性为GONE的控件仍然是其中的一部分，具有重要的区别：

  • 对于布局传递，它们的维度将被视为零（基本上，它们将被解析为一个点）
  • 如果他们对其他控件有约束，他们仍然会被考虑到，但是任何侧边距都将等于零

这种特定行为允许构建布局，可以暂时将控件标记为GONE，而不会破坏布局（图7），这在执行简单的布局动画时特别有用。

注意：使用的边距将是B连接到A时定义的边距（例如参见图7）。在某些情况下，这可能不是想要的边距（例如，A对其容器一侧有100dp的边距，B对A只有一个16dp，将A标记为GONE，B对容器会有16dp的边距）。由于这个原因，当连接到一个被标记为GONE的控件时，可以指定一个可选的的边距。

尺寸约束（Dimensions constraints）

ConstraintLayout的最小尺寸

可以为ConstraintLayout本身定义最小和最大尺寸：

  • android:minWidth 为布局设置最小宽度
  • android:minHeight 为布局设置最小高度
  • android:maxWidth 为布局设置最大宽度
  • android:maxHeight 为布局设置最大高度

当尺寸设置为WRAP_CONTENT时，ConstraintLayout将使用这些最小尺寸和最大尺寸。

控件尺寸约束

可以通过以3种不同的方式设置android:layout_width和android:layout_height属性来指定控件的尺寸：

  • 使用特定尺寸 (固定值或尺寸的引用)
  • 使用WRAP_CONTENT, 这会要求控件计算自己的大小
  • 使用0dp，相当于MATCH_CONSTRAINT

重要提示：不建议将MATCH_PARENT用于ConstraintLayout中包含的控件。类似的行为可以通过使用MATCH_CONSTRAINT来定义，相应的left/right或top/bottom约束被设置为parent

WRAP_CONTENT：强制约束（在1.1中添加）

如果尺寸设置为WRAP_CONTENT，则在1.1之前的版本中，将被视为字面量尺寸 - 这意味着约束不会限制生成的尺寸。虽然通常这已经足够（并且更快），但在某些情况下，可能需要使用WRAP_CONTENT，但仍需执行约束来限制生成的尺寸。在这种情况下，可以添加相应的属性：

  • app:layout_constrainedWidth=”true|false”
  • app:layout_constrainedHeight=”true|false”

MATCH_CONSTRAINT尺寸（在1.1中添加）

当尺寸设置为MATCH_CONSTRAINT时，默认行为是使得生成的大小占用所有可用空间。有几个额外的修饰符可用：

  • layout_constraintWidth_min和layout_constraintHeight_min : 会设置最小尺寸大小
  • layout_constraintWidth_max和layout_constraintHeight_max : 会设置最大尺寸大小
  • layout_constraintWidth_percent和layout_constraintHeight_percent : 会将此尺寸的大小设置为父级的百分比

最小和最大

min和max指示的值可以是Dp中的一个尺寸，也可以是wrap，它将使用与WRAP_CONTENT相同的值。

百分比尺寸

要使用百分比，需要设置以下内容：

  • 尺寸应该设置为MATCH_CONSTRAINT (0dp)
  • 默认值应设置为百分比app:layout_constraintWidth_default=”percent”或applayout_constraintHeight_default=”percent”
    （注意：这在1.1-beta1和1.1-beta2中是必需的，但如果定义percent属性，则在以下版本中不需要）
  • 然后将layout_constraintWidth_percent或layout_constraintHeight_percent属性设置为介于0和1之间的值

比值（Ratio）

还可以将控件的一个尺寸定义为另一个尺寸的比例。因此至少需要将一个约束尺寸设置为0dp（即，MATCH_CONSTRAINT），并将属性layout_constraintDimensionRatio设置为给定比率。

将按钮的高度设置为与其宽度相同。

比例可以表示为：

  • 浮点值，表示宽度和高度之间的比率
  • “width:height”形式的比率

如果两个尺寸都设置为MATCH_CONSTRAINT（0dp），也可以使用比率。在这种情况下，系统设置满足所有约束条件的最大尺寸并保持指定的宽高比。为了约束一个特定的边，基于另一个边的尺寸，可以预先附加W或H，分别约束宽度或高度。例如，如果一个尺寸受到两个目标的约束（例如，宽度为0dp并以父控件为中心），可以通过在比率前添加字母W（用于限制宽度）或H（用于限制高度），以逗号分隔来指示应该约束哪一侧：

链（Chains）

链在单个轴（水平或垂直）中提供类似组的行为。另一个轴可以独立约束。

创建链条（Creating a chain）

如果一组控件通过双向连接在一起，则它们被视为链。

链条头（Chain heads）

链由链的第一个元素（链的“头”）上设置的属性控制：

头是水平链最左边的部件，也是垂直链最顶端的部件。

链的边距（Margins in chains）

如果在连接上指定了边距，将被考虑在内。在扩展链的情况下，边距将从分配的空间中减去。

链式（Chain Style）

在链的第一个元素上设置属性layout_constraintHorizontal_chainStyle或layout_constraintVertical_chainStyle时，链的行为将根据指定的样式进行更改（默认为CHAIN_SPREAD）

  • CHAIN_SPREAD - 元素将被展开（默认样式）
  • 加权链 - 在CHAIN_SPREAD模式下，如果某些控件设置为MATCH_CONSTRAINT，则它们将拆分可用空间
  • CHAIN_SPREAD_INSIDE - 类似，但链的端点不会被扩展
  • CHAIN_PACKED - 链的元素将被打包在一起。孩子的水平或垂直偏移属性将影响打包元素的位置

加权链（Weighted chains）

链的默认行为是在可用空间中均匀分布元素。如果一个或多个元素使用MATCH_CONSTRAINT，则它们将使用可用的空白空间（在它们之间平均分配）。属性layout_constraintHorizontal_weight和layout_constraintVertical_weight将控制空间将如何在使用MATCH_CONSTRAINT的元素之间分配。例如，在包含使用MATCH_CONSTRAINT的两个元素的链中，第一个元素的权重为2，第二个权重为1，第一个元素占用的空间将是第二个元素的两倍。

边距和链（Margins and chains） (在1.1中添加)

在链中的元素上使用边距时，边距将会：

  • 叠加：例如在水平链上，如果一个元素定义了10dp的右边距，而下一个元素的左边距为5dp，则这两个元素之间的结果边距将为15dp
  • 链条用来放置元素的剩余空间不包含边距（可以认为它的位置的是元素加边距）

Guideline

表示ConstraintLayout的指导辅助对象的实用工具类。辅助对象不会显示在设备上（它们被标记为View.GONE），仅用于布局目的。他们只能在ConstraintLayout中工作。

Guideline可以是水平的或垂直的：

  • 垂直Guideline的宽度为零，高度为ConstraintLayout的高度
  • 水平Guideline的高度为零，宽度为ConstraintLayout的宽度

定位Guideline有三种不同的方式：

  • 指定布局左侧或顶部的固定距离（layout_constraintGuide_begin）
  • 从布局的右侧或底部指定固定距离（layout_constraintGuide_end）
  • 指定布局的宽度或高度的百分比（layout_constraintGuide_percent）

然后可以将控件约束到Guideline，允许从一个Guideline轻松定位多个控件，或者通过使用百分比定位来允许响应式布局行为。

多个组（Multiple groups）

多个组可以引用相同的控件 - 在这种情况下，XML声明顺序将定义最终的可见性状态（最后声明的组将具有最终话语权）。

Placeholder（在1.1中添加）

占位符提供了一个可以定位现有对象的虚拟对象。

当另一个视图的id在占位符（使用setContent()）上设置时，占位符有效地变为内容视图。如果内容视图存在于屏幕上，则视为从其原始位置消失。

内容视图使用占位符参数的布局进行定位（占位符像任何其他视图一样仅限制在布局中）。

ConstraintSet

这个类允许以编程的方式定义一组ConstraintLayout使用的约束。它允许创建和保存约束，并将它们应用于现有的ConstraintLayout。ConstraintsSet可以以各种方式创建：

  • 手动（Manually）
    c = new ConstraintSet(); c.connect(….);
  • 从R.layout.*对象
    c.clone(context, R.layout.layout1);
  • 从ConstraintLayout
    c.clone(clayout);

优化（Optimizer）（在1.1中添加）

在1.1中公开了约束优化器。可以通过将标签app:layout_optimizationLevel添加到ConstraintLayout元素来决定应用哪些优化。

  • none : no optimizations are applied
  • standard : Default. Optimize direct and barrier constraints only
  • direct : optimize direct constraints
  • barrier : optimize barrier constraints
  • chain : optimize chain constraints (experimental)

此属性是一个掩码，因此可以决定打开或关闭特定的优化，方法是列出所需的优化。例如：app:layout_optimizationLevel=”direct|barrier|chain”
