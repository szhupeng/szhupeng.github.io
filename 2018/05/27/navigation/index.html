新架构组件：Navigation详解

发表于 2018-05-27 | 分类于 Android | 阅读次数:

Android Gradle

dependencies {
  def nav_version = "1.0.0-alpha02"

  implementation "android.arch.navigation:navigation-fragment:$nav_version" // use -ktx for Kotlin
  implementation "android.arch.navigation:navigation-ui:$nav_version" // use -ktx for Kotlin
}

对于安全参数，请在顶级build.gradle文件中添加以下类路径：

buildscript {
  repositories {
      google()
  }
  dependencies {
      classpath "android.arch.navigation:navigation-safe-args-gradle-plugin:1.0.0-alpha02"
  }
}

并在应用程序或模块build.gradle文件中：

apply plugin: "androidx.navigation.safeargs"

遵循原则

任何应用内导航的目标应该是为用户提供一致且可预测的体验。为了实现这个目标，Navigation架构组件可以帮助构建一个遵循以下每个导航原则的应用程序。

应用应该有一个固定的起始目的地

应用程序有一个固定的目的地，即用户从启动器启动应用程序时看到的屏幕。这个目的地也应该是用户在按下后退按钮后返回到启动程序时看到的最后一个屏幕。

注意：应用程序可能有一次设置或一系列登录的屏幕。这些带条件的屏幕不应被视为应用程序的起始目标。

堆栈用于表示应用程序的“导航状态”

应用的导航状态应使用后进先出结构表示。这个“导航堆栈”应该在堆栈的底部有应用程序的起始目标，并且当前目标是堆栈的顶部。

应始终操作导航堆栈顶部来改变导航堆栈，或者将新目标“推入”堆栈顶部或从堆栈中“弹出”最顶部的目标。

向上按钮从不退出应用程序

如果用户在起始目的地，则不应显示向上按钮。当应用程序使用其他应用程序任务的深层链接启动时，Up应该将用户带到分层父目标，而不是返回到其他应用程序。

Up和Back在应用程序的任务中相当

当系统“后退”按钮不会退出应用程序时，例如当在自己的任务上而不在起始目标上时，向上按钮的功能应与系统“后退”按钮相同。

深度链接到目标或导航到同一目标应产生相同的堆栈

用户可以在起始目标进入应用程序并导航到目的地。用户还可以使用深层链接（如果可用）导航到同一目的地。在这两种情况下，导航堆栈应具有相同的目标堆栈。具体而言，无论用户如何到达目的地，用户都应该能够使用“返回”或“向上”按钮来将目标导航回开始目的地。删除任何现有导航堆栈并替换为深层链接的导航堆栈。

实现

导航架构组件简化了应用程序中目标之间导航的实现。目的地是应用中的特定屏幕。默认情况下，导航架构组件包括支持fragment和activity作为目标，但也可以添加对新目标类型的支持。一组目的地组成一个应用程序的“导航图”。

除目的地之外，导航图在目标之间有连接，称为“actions”。图1显示了包含6个目的地的示例应用程序的导航图的直观表示，该应用程序由5个操作连接起来。

导航架构组件基于导航原则实现。

设置

在创建导航图之前，必须为项目设置导航架构组件。要在Android Studio中设置项目，请执行以下步骤。

  1. 将导航架构组件添加到应用程序或模块的build.gradle文件中。
  2. 在Project窗口中，右键单击res目录并选择New>Android资源文件。出现新资源对话框。
  3. 在“文件名”字段中输入名称，例如“nav_graph”。
  4. 从资源类型下拉列表中选择导航。
  5. 点击确定。发生以下情况：
    • 在res目录中创建navigation资源目录。
    • 在navigation目录中创建nav_graph.xml文件。
    • nav_graph.xml文件将在导航编辑器中打开。这个XML文件包含导航图。
  6. 点击文本标签切换到XML文本视图。空导航图的XML如下所示：
  7. 点击Design返回导航编辑器。

浏览导航编辑器

导航编辑器的部分是：

  • 目的地列表 - 列出当前在图表编辑器中的所有目的地。
  • 图表编辑器 - 包含导航图的可视化表示。
  • 属性编辑器 - 包含与导航图中的目标和操作关联的属性。

识别目的地

创建导航图的第一步是确定应用的目的地。可以在现有项目中创建空白目的地或创建fragment和activity的目的地。

注意：导航架构组件专为一个主activity具有多个fragment目标的的应用程序而设计。主activity“托管”导航图。在具有多个activity目标的应用中，每个附加activity都会托管自己的导航图。

要识别应用的目标，请使用以下步骤。

  1. 在“图表编辑器”中，单击“New Destination”出现新目标对话框。
  2. 单击“创建空白目标”或单击fragment或activity。将出现“新建Android组件”对话框。
  3. 在“Fragment Name”字段中输入名称。此名称是fragment类的名称。
  4. 在“Fragment Layout Name”字段中输入名称。该名称是该fragment的布局文件的名称。
  5. 单击完成。代表目标的框出现在图形编辑器和目标列表中。发生以下情况：
    • 如果创建了空白目的地，图表编辑器会在目的地显示消息“Hello blank fragment”。如果单击fragment或activity，图表编辑器会显示该activity或fragment的布局预览。
    • 为项目创建一个Fragment子类。此类具有在步骤3中指定的名称。
    • 为项目创建资源文件。该文件具有在步骤4中分配的名称。
  6. 点击新插入的目的地以突出显示目的地。“属性”面板中显示以下属性：
    • “Type”字段包含“Fragment”或“Activity”，以指示目标是否在源代码中实现为fragment或activity。
    • “Label”字段包含目标的XML布局文件的名称。
    • “ID”字段包含将用于引用代码中的目的地的ID。
    • “Class”字段包含目标类的名称。
  7. 单击“文本”选项卡以切换到XML视图。XML现在包含基于现有类和布局文件的名称的id，名称（类名），标签和布局属性：

注意：XML具有指向新空目标的id的startDestination属性（app:startDestination=”@+id/fragment”）。

以类型安全的方式在目标之间传递数据

导航架构组件有一个Gradle插件，称为safeargs，它生成简单的对象和构建器类，以便对目标和操作指定的参数进行类型安全访问。Safe args建立在Bundle方法的基础之上，但需要一些额外的代码来换取更多类型的安全性。如果使用的是Gradle，则可以使用safe args插件。要添加此插件，请将“androidx.navigation.safeargs”插件添加到build.gradle文件中。例如：

apply plugin: 'com.android.application'
apply plugin: 'androidx.navigation.safeargs'

android {
  //...
}

配置Gradle插件后，请按照以下步骤使用类型安全的args。

  1. 在Graph Editor器中，单击接收参数的目标位置。目的被选中。
  2. 单击“Attributes”面板的“Arguments”部分中的+。出现空名称和默认值字段。
  3. 双击名称并输入参数的名称。
  4. 按Tab键并从下拉列表中选择参数的类型。
  5. 按Tab键并输入参数的默认值。
  6. 单击此目标之前的操作。参数默认值应包含新添加的参数。
  7. 单击“文本”选项卡以切换到XML视图。带有name和defaultValue属性的参数元素已添加到目标：

使用safeargs插件生成代码时，会为操作以及发送和接收目标创建简单对象和构建器类。这些类是：

  • 动作源自的目的地的类，附加“Directions”一词。
    因此，如果原始fragment标题为SpecifyAmountFragment，则生成的类称为SpecifyAmountFragmentDirections。这个类有一个方法，以用于传递参数的动作命名，用于捆绑参数，例如confirmationAction()。
  • 一个内部类，其名称基于用于传递参数的操作。如果传递的操作名为confirmationAction，则该类名为ConfirmationAction。
  • 传递参数的目标的类，附加单词Args。
    因此，如果目标fragment标题为ConfirmationFragment，则生成的类称为ConfirmationFragmentArgs。使用此类的fromBundle()方法来检索参数。

将目标分组为嵌套导航图

可以将一系列目的地分组为导航图中的子图。子图称为“嵌套图”，而包含图称为“根图”。嵌套图对于组织和重用应用程序UI的各个部分非常有用，例如单独的登录流程。

与根图一样，嵌套图必须将目标标识为起始目标。嵌套图形封装了它的目的地；嵌套图外部的目标（例如根图上的目标）仅通过其起始目标访问嵌套图。图6显示了简单汇款应用程序的导航图。该图有两个流程：允许用户汇款的流程和允许用户查看其余额的流程。

  2. 打开上下文菜单，然后选择“Move to Nested Graph > New Graph”。目标包含在嵌套图中。
  3. 单击嵌套图以突出显示它。“Attributes”面板中显示以下属性：

为目标创建深层链接

在Android中，深层链接是指向应用中特定目标的URI。当希望将用户发送到特定目的地以执行某项任务时，这些URI非常有用，例如允许用户快速向某人汇款的汇款流程。

为目标分配深层链接

要在导航图中为目标分配深层链接，请执行以下操作：

  1. 从Graph Editor中，选择深层链接的目标。
  2. 单击“Attributes”面板的“Deep Links”部分中的+。将出现“Add Deep Link”对话框。
  3. 在URI字段中键入URI，例如“www.cashdog.com/sendmoney”，它表示应用程序中发送货币嵌套图的起始目的地。
    请注意以下事项：
    • 没有scheme的URI被认定为http和https。例如，www.cashdog.com与http://www.cashdog.com和https://www.cashdog.com相匹配。
    • 占位符以{placeholder_name}的形式匹配1个或多个字符。占位符的String值在参数Bundle中可用，并带有相同名称的键。例如，http：//www.example.com/users/{id}与http://www.example.com/users/4相匹配。
    • *通配符可用于匹配0个或多个字符。
  4. （可选）选中“Check Auto Verify”以要求Google验证您是URI的所有者。
  5. 点击Add。所选目标上方会显示一个链接图标，表示该目标具有深层链接。
  6. 单击“文本”选项卡以切换到XML视图。已将嵌套的深层链接元素添加到目标：

当用户使用深层链接目标中的“后退”按钮时，他们会导航回导航堆栈，就像他们从应用程序的入口点进入您的应用程序一样。

在目的地之间创建过渡

导航架构组件提供了在目标之间轻松添加转换（例如淡入和淡出）的功能。要添加转换：

  1. 创建动画资源文件。导航架构组件支持属性和视图动画。
  2. 在“Graph Editor”中，单击应进行过渡的操作。
  3. 在“Attributes”面板的“Transitions”部分中，单击“Enter”旁边的向下箭头。将显示项目中的转换列表。
  4. 选择用户进入目的地时发生的转换。
  5. 在“Attributes”面板的“Transitions”部分中，单击“Exit”旁边的向下箭头。将显示项目中的转换列表。
  6. 选择用户退出目标时发生的转换。
  7. 单击“文本”选项卡以切换到XML文本视图。转换的XML出现在步骤2中指定的操作的action元素中。该操作嵌入在转换发生之前处于活动状态的目标的XML中。 在以下示例中，specifyAmountFragment是活动目标，它包含具有过渡动画的操作： 在此示例中，在移动到目标时会发生转换（enterAnim和exitAnim以及退出该目标时（popEnterAnim和popExitAnim））。

迁移到导航架构组件

NavController及其导航图包含在单个activity中。因此，在迁移现有项目以使用导航体系结构组件时，请通过为每个activity中的目标创建导航图，重点关注一次迁移一个activity。

然后可以通过向导航图添加activity目标来链接单独的活动，在整个代码库中替换startActivity()的现有用法。

在多个activity共享相同布局的情况下，可以组合导航图，将导航调用替换为活动目标，以直接在两个导航图之间导航调用。
